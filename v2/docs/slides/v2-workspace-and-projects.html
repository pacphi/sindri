Workspace Structure

Where Everything Lives

The Sindri Workspace

$HOME/ # /alt/home/developer ├── workspace/ # $WORKSPACE - main working directory │ ├── projects/ # Your code projects ($WORKSPACE_PROJECTS) │ │ ├── my-api/ # Individual project │ │ ├── web-frontend/ # Another project │ │ └── ... │ ├── config/ # User configurations │ ├── scripts/ # Custom scripts │ ├── bin/ # Local binaries (in PATH) │ └── .system/ # Extension state (manifest, logs) ├── .local/share/mise/ # Tool installations (Node, Python, etc.) ├── .config/ # XDG configuration └── .cache/ # Cached data

Key Environment Variables

Variable Path Purpose
$HOME /alt/home/developer User home directory
$WORKSPACE $HOME/workspace Main working directory
$WORKSPACE_PROJECTS $WORKSPACE/projects Where projects live
$DOCKER_LIB /docker/lib Immutable system files

Projects are created in $WORKSPACE_PROJECTS by default

Persistent Storage

  • Volume-based - All of $HOME survives restarts
  • Tool installations - mise-managed tools persist
  • Project data - Your code is always there
  • Git config - User settings remembered
  • Extension state - Manifests and logs retained

Stop and restart your container - your work is safe!

new-project

Create Projects from Templates

Auto-detect

11 Templates

Claude Enhanced

The Value Proposition

  • Intelligent Detection - Guesses project type from name
  • Template-based Setup - Proper structure from the start
  • Extension Activation - Installs required tools automatically
  • Claude Integration - Creates CLAUDE.md for AI context
  • Git Initialization - Ready for version control immediately

Go from idea to coding in seconds, not minutes

Basic Usage

# Auto-detect project type from name
new-project my-rails-app        # Detects "rails" from name
new-project api-service         # Prompts - multiple types match "api"
new-project python-analyzer     # Detects "python" from name

# Specify type explicitly
new-project my-app --type node
new-project backend --type go
new-project cloud-infra --type terraform

# List available templates
new-project --list-types

What Gets Created

$ new-project my-node-api --type node

# Creates:
my-node-api/
├── .git/                   # Git repository initialized
├── .gitignore              # Language-appropriate ignores
├── package.json            # With npm scripts configured
├── CLAUDE.md               # AI context for Claude
└── (initial commit made)

# Also:
# - nodejs extension activated
# - Git user configured
# - Ready for immediate development

Command Options

new-project <name> [options]

Options:
  --type <type>       Specify project type explicitly
  --list-types        Show all available project types
  --interactive       Force interactive type selection
  --git-name <name>   Set Git user name for this project
  --git-email <email> Set Git user email for this project
  -h, --help          Show help message

clone-project

Clone & Enhance Existing Repos

The Value Proposition

  • Fork Support - Fork and clone in one command
  • Claude Enhancements - CLAUDE.md and Git hooks
  • Dependency Detection - Auto-install project deps
  • Feature Branches - Create branch immediately
  • Per-Project Git Config - Different identity per repo

Clone any repo and it's ready for AI-assisted development

Basic Usage

# Simple clone with enhancements
clone-project https://github.com/user/awesome-project

# Fork, then clone (requires gh CLI)
clone-project https://github.com/org/project --fork

# Clone and create feature branch
clone-project https://github.com/user/repo --feature add-auth

# Full workflow: fork + feature branch
clone-project https://github.com/org/repo --fork --feature my-feature

What Gets Added

$ clone-project https://github.com/user/api-server

# Clones to:
$WORKSPACE_PROJECTS/api-server/

# Adds:
├── CLAUDE.md               # AI context (if not present)
├── .git/hooks/             # Pre-commit, commit-msg hooks
└── (dependencies installed if detected)

# Output:
Project api-server cloned successfully
Location: /alt/home/developer/workspace/projects/api-server
Next steps:
   1. cd api-server
   2. Start coding with: claude

Command Options

clone-project <repository-url> [options]

Options:
  --fork              Fork repo before cloning (requires gh CLI)
  --branch <name>     Checkout specific branch after clone
  --depth <n>         Shallow clone with n commits
  --git-name <name>   Configure Git user name for this project
  --git-email <email> Configure Git user email for this project
  --feature <name>    Create and checkout feature branch
  --no-deps           Skip dependency installation
  --no-enhance        Skip all enhancements (just clone/fork)
  -h, --help          Show help message

Fork Workflow

# Complete contribution workflow
clone-project https://github.com/org/project \
  --fork \
  --feature add-dark-mode \
  --git-name "Your Name" \
  --git-email "you@example.com"

# Sets up:
# - origin: your fork
# - upstream: original repo
# - Branch: add-dark-mode
# - Git aliases for sync: git sync-upstream

Project Templates

11 Built-in Templates

Available Templates

node python go rust rails django spring dotnet terraform docker general

Each template includes: files, .gitignore, extensions, setup commands, and CLAUDE.md template

Template Details: Node.js

node:
  description: "Node.js application"
  aliases: ["nodejs", "javascript"]
  extensions:
    - nodejs
  detection_patterns: ["node", "npm", "express", "js", "typescript", "ts"]
  setup_commands:
    - "npm init -y"
  dependencies:
    detect: "package.json"
    command: "npm install"
  files:
    "package.json": |
      {
        "name": "{project_name}",
        "scripts": { "start": "node index.js", ... }
      }
    ".gitignore": "node_modules/\ndist/\n..."

Template Details: Python

python:
  description: "Python application"
  aliases: ["py", "python3"]
  extensions:
    - python
  detection_patterns: ["python", "py", "django", "flask", "fastapi", "ml"]
  setup_commands:
    - "python3 -m venv venv"
    - "source venv/bin/activate"
    - "touch requirements.txt"
  dependencies:
    detect: "requirements.txt"
    command: "pip3 install -r requirements.txt"
  files:
    "main.py": "#!/usr/bin/env python3\n..."
    ".gitignore": "__pycache__/\nvenv/\n..."

Template Details: Terraform

terraform:

  aliases: ["tf", "infra", "infrastructure"]
  extensions:

Custom Templates

Add Your Own Templates

Template Configuration File

# Templates are defined in:
/docker/lib/project-templates.yaml

# Schema validation:
/docker/lib/schemas/project-templates.schema.json

# Helper functions:
/docker/lib/project-templates.sh

Templates are YAML-driven - add new templates by editing project-templates.yaml

Template Structure

templates:
  mytemplate:
    description: "My custom template"
    aliases: ["mt", "custom"]           # Optional short names
    extensions:                          # Extensions to install
      - nodejs
      - docker
    detection_patterns:                  # Name patterns to match
      - "mypattern"
      - "custom"
    setup_commands:                      # Run after project creation
      - "npm init -y"
    dependencies:                        # Auto-install dependencies
      detect: "package.json"
      command: "npm install"
    files:                               # Files to create
      "README.md": "# {project_name}"
      ".gitignore": "node_modules/"
    claude_md_template: |                # CLAUDE.md content
      # {project_name}
      ## Setup
      ...

Template Variables

Use these placeholders in files and claude_md_template:

Variable Value
{project_name} Name of the project
{author} Git user name
{git_user_name} Git user name
{git_user_email} Git user email
{date} Current date (YYYY-MM-DD)
{year} Current year

Example: Adding a FastAPI Template

fastapi:
  description: "FastAPI Python web application"
  aliases: ["fast", "fastapi-app"]
  extensions:
    - python
  detection_patterns: ["fastapi", "fast", "api-py"]
  setup_commands:
    - "python3 -m venv venv"
    - "source venv/bin/activate"
    - "pip install fastapi uvicorn"
  dependencies:
    detect: "requirements.txt"
    command: "pip install -r requirements.txt"
  files:
    "main.py": |
      from fastapi import FastAPI
      app = FastAPI()

      @app.get("/")
      def read_root():
          return {"Hello": "World"}
    "requirements.txt": |
      fastapi>=0.100.0
      uvicorn>=0.23.0

Detection Rules

detection_rules:
  # Name pattern matching (regex)
  name_patterns:
    - pattern: ".*-?rails?-?.*"
      type: "rails"
    - pattern: ".*-?api.*"
      types: ["node", "go", "python"]    # Prompts user to choose
    - pattern: ".*-?service.*"
      types: ["go", "node", "spring"]

  # Framework keywords
  framework_keywords:
    rails: ["rails", "ruby"]
    django: ["django", "python"]
    spring: ["spring", "java", "kotlin"]

Add detection rules to enable auto-detection for your custom templates

Validation

# Validate your template changes
pnpm validate:yaml

# Or manually with yamllint
yamllint docker/lib/project-templates.yaml

# Schema validation happens automatically
# when templates are loaded

Templates are validated against project-templates.schema.json
